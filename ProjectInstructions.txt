Self-made fast look-up guide for the project.

Services (Srv): security (sec), health (hlt) and education (edu)

Utilities (Utl): electricity (elc), water (wtr) and internet (int)

Resources (Rsr): population (ppl), goods (gds), lifestyle (lfs)

------------------------------------------------------------------------------------
Zones:
    Srv Zones: Works within a range, connections are not necessary
        Police Station (F): +sec
        Hospital (D): +hlt
        School (S): +edu

    Utl Zones: They operate per tick
        Power Plant (P): +100 elc
        Water Pumping Station (W): +100 wtr
        Internet Hub (T): +100 int

    Rsr Zones: They start at lvl 0
        House (H): Utl -> ppl
        Industrial (I): Utl + ppl -> gds
        Commercial (C): Utl + ppl + gds -> lfs

    Extras:
        Road (R): Connection, transfers Rsr + Utl
        Empty Cell (E): Nothing

------------------------------------------------------------------------------------

Distribution (Dist):
    Srv Dist: Boolean, zones either has it or not. Use Euclidean or Manhattan to calculate hypotenuse.
    Utl Dist: From an Utl zone, all adjacent zones (except E) will receive Utl as much as demands, Roads will connect.
    Rsr Dist: All Rsr shared equally on any zone that will use them. No connection needed.

------------------------------------------------------------------------------------

Levels:
    H: Utl -> lvl 1 + sec + hlt -> lvl 2 + lfs -> lvl 3
    I: ppl + elc + wtr -> lvl 1 + sec -> lvl 2 + excess ppl -> lvl 3
    C: ppl + gds + Utl -> lvl 1 + sec -> lvl 2 + excess ppl + excess gds -> lvl 3

------------------------------------------------------------------------------------

Outputs:        LVL           0               1               2               3
    H:          ppl           0               m              2m            2m + lfs as much as received
    I:          gds           0               m              2m            2m + ppl as much as received
    I:          lfs           0               m              2m            2m + min(ppl, gds) received

------------------------------------------------------------------------------------

Utility Demands:
    A zones demand for the next tick equals to the output of it in the current tick (min 1).
    So basically a zone that produces more also demands more to protect that output.
    This causes zones closer to the Utl zones to get more Utl from them while further ones receive less.

------------------------------------------------------------------------------------

Simulation: Each tick will run at this procedure. You have to provide map ad tick for simulation.
    Step 1: Srv zones applied on other zones within that range.
    Step 2: Utl Dist via BFS (breadth first search).
    Step 3: Rsr production from previous tick distributed (tick 0 = 0 Rsr).
    Step 4: Rsr zones (H, I, C) updated. Each zone computes its new lvl and output based on the last tick.
    Step 5: All production accumulated (summed up) for the distribution (division) on next tick.

    In the first tick, simulation sets itself, steps are applied but there is no production yet. From the tick 2 and
    onwards; H will produce ppl to enable and run I and C, they will produce Rsr and H will lvl up. At least that's
    what I understand. So the key idea here:

    H -> ppl -> I & C -> gds + lfs -> H +lvl -> LOOP
